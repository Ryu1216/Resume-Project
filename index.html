UI 디자인 시스템 프론트엔드 개발 데이터 시각화 대시보드

문제 정의

복잡한 데이터를 직관적으로 파악할 수 있는 대시보드 UI가 필요했습니다. 기존 시스템은 사용자가 많은 클릭을 해야 정보에 접근할 수 있었고, 데이터 간 연관성을 파악하기 어려웠습니다.

사용자 중심 접근

실제 데이터 분석가와 매니저들을 대상으로 인터뷰를 진행하여 일상 업무에서 가장 자주 확인하는 지표와 의사결정 패턴을 파악했습니다. 이를 통해 핵심 지표를 한눈에 볼 수 있는 대시보드 레이아웃을 설계했습니다.

데이터 기반 디자인

히트맵 분석을 통해 사용자의 시선이 가장 많이 머무는 영역을 파악하고, 중요 지표를 해당 위치에 배치했습니다. 또한 색상 코드를 통해 데이터의 변화량과 중요도를 직관적으로 인식할 수 있도록 했습니다.

협업 과정

백엔드 개발자와 정기적인 미팅을 통해 데이터 구조를 이해하고, 최적의 시각화 방법을 함께 고민했습니다. 프론트엔드 개발자와는 컴포넌트 단위로 UI를 설계하여 재사용성과 일관성을 높였습니다.

배운 점

이 프로젝트를 통해 복잡한 데이터를 단순화하면서도 정보의 가치를 유지하는 방법을 배웠습니다. 특히 사용자가 어떤 맥락에서 데이터를 소비하는지 이해하는 것이 효과적인 데이터 시각화의 핵심임을 깨달았습니다.

CineAI 영화 추천 서비스

팀 프로젝트

2025.02-2025.03

UX 디자인 프론트엔드 개발 웹 페이지 영화 추천

문제 정의

AI 기반 영화 추천 알고리즘의 결과를 사용자에게 어떻게 효과적으로 전달할 것인가가 핵심 과제였습니다. 단순히 추천 목록을 나열하는 것이 아닌, 왜 이 영화가 추천되었는지 이해할 수 있어야 했습니다.

사용자 중심 접근

영화 애호가들을 대상으로 사용자 조사를 실시하여 영화 선택 시 고려하는 주요 요소들(감독, 배우, 장르, 분위기 등)을 파악했습니다. 이를 바탕으로 추천 알고리즘의 결과를 사용자가 이해하기 쉬운 형태로 시각화했습니다.

데이터 기반 디자인

사용자 행동 데이터를 분석하여 가장 클릭률이 높은 UI 요소와 레이아웃을 파악했습니다. A/B 테스트를 통해 '찜하기' 버튼의 위치와 디자인을 최적화하여 사용자 상호작용을 20% 증가시켰습니다.

협업 과정

AI 개발자와 긴밀하게 협력하여 추천 알고리즘의 작동 방식을 이해하고, 이를 사용자에게 투명하게 전달할 수 있는 UI를 설계했습니다. 프론트엔드 개발자와 함께 프로토타입을 제작하고 사용성 테스트를 통해 지속적으로 개선했습니다.

배운 점

이 프로젝트에서 AI와 같은 복잡한 기술을 사용자가 이해하기 쉽게 시각화하는 것의 중요성을 배웠습니다. 또한 팀원 간의 효과적인 커뮤니케이션이 프로젝트 성공의 핵심임을 실감했습니다.

하루한 뉴스레터 서비스 와이어프레임

팀 프로젝트

2025.01-2025.04

와이어프레이밍 사용자 플로우 모바일 UX 프론트엔드 개발

문제 정의

사용자들이 앱에 처음 진입했을 때 겪는 혼란과 높은 이탈률이 주요 문제였습니다. 복잡한 기능을 어떻게 직관적으로 소개하고 온보딩할 것인지가 핵심 과제였습니다.

사용자 중심 접근

초기 단계에서 사용자 인터뷰를 통해 앱에 대한 기대와 우려사항을 파악했습니다. 다양한 연령층과 디지털 숙련도를 고려하여 누구나 쉽게 이해할 수 있는 단계별 온보딩 프로세스를 설계했습니다.

데이터 기반 디자인

페이퍼 프로토타입으로 초기 사용성 테스트를 진행하고, 사용자의 피드백을 정량적으로 분석했습니다. 이를 바탕으로 사용자 여정 지도를 작성하고 각 단계별 마찰 포인트를 식별하여 개선했습니다.

협업 과정

팀원과의 정기적인 회의를 통해 뉴스레터 서비스의 비즈니스 요구사항과 기술적 제약을 이해하고, 이를 고려한 현실적인 디자인 솔루션을 제시했습니다. 다양한 이해관계자의 의견을 수렴하고 절충점을 찾는 과정에서 커뮤니케이션 능력을 향상시켰습니다.

배운 점

이 프로젝트를 통해 와이어프레임이 단순한 레이아웃 툴이 아닌, 다양한 이해관계자들과의 커뮤니케이션 도구로서 중요한 역할을 한다는 것을 배웠습니다. 또한 초기 단계에서의 철저한 사용자 리서치가 이후 디자인 과정에서 발생할 수 있는 수많은 시행착오를 방지한다는 교훈을 얻었습니다.

모디브UI (2025.04)

이전 프로젝트들에서의 피드백 수용 과정을 통해 더 체계적인 디자인 시스템을 구축하게 되었습니다. 데이터 시각화라는 새로운 도전에 직면하면서, 정보의 계층화와 직관적인 인터랙션 설계 능력이 크게 향상되었습니다.

세 가지 프로젝트를 거치며, 단순한 UI 요소 배치에서 시작해 사용자 니즈와 비즈니스 목표를 통합하는 전략적 UX 플래너로 성장해왔습니다. 매 프로젝트마다 '왜'라는 질문을 통해 디자인 결정의 근거를 마련하고, 피드백을 통해 지속적으로 발전하는 과정이 제 성장의 핵심이었습니다.
